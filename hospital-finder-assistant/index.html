Hospital Finder Assistant

A voice- and text-enabled conversational AI that helps users find hospitals by location, specialty and insurance — combining speech-to-text, LLM intent recognition, FAISS semantic retrieval (RAG), a QLoRA fine-tuned model, and text-to-speech, orchestrated with LangGraph.

Python 3.11 LangGraph FAISS RAG QLoRA / PEFT OpenAI Whisper + TTS Gemini 2.0 Flash Pony ORM + SQLite spaCy NER

❓ ClarifyIf info missing

→
✨ GroundLLM / QLoRA
→
🔊 RespondTTS reply

The six intents

  • find_nearest — nearest hospitals by distance
  • find_best — top-rated hospitals
  • find_by_insurance — hospitals covered by a provider
  • find_by_hospital — details about a specific hospital
  • compare_hospitals — compare two or more facilities
  • exit — end the conversation

Design highlights

  • LangGraph state machine — multi-turn dialogue with clarification loops
  • Hybrid recognition — LLM-based or spaCy NER + fuzzy matching
  • RAG grounding — semantic search filtered by Haversine distance & rating
  • QLoRA fine-tune — 4-bit Open-LLaMA-3B with LoRA adapters for hospital queries
  • Pluggable models — STT, LLM, TTS and lookup mode all configurable

Run It in 4 Steps

# 1 — clone
git clone https://github.com/navinpandiyan/hospital-finder-assistant.git
cd hospital-finder-assistant

# 2 — install (uv handles the venv + deps)
pip install uv
uv sync

# 3 — add your keys to a .env file
OPENAI_API_KEY="sk-..."      # Whisper STT, embeddings, TTS
LLM_API_KEY="..."           # recognition / clarify / RAG grounding LLM
LLM_BASE_URL="https://openrouter.ai/api/v1"

# 4 — run (first run seeds the DB, builds FAISS, optionally fine-tunes)
uv run app.py

Default mode is chatbot (text). Set MODE = "voicebot" in settings/config.py for full speech in/out. Full details in the Setup & Run Guide.
